Series — 6 design variations

Each variation is built from existing CMS components (view-header, content-block, cms-table, rowform-row, pill, button family) so they read as the same software. Same sample data across all six: three series with mixed counts and a mix of stages.

V1 — Table list
Series as rows in the canonical list table

Treat each series like an article: a row in .cms-table. Click a row to navigate to a per-series edit page (separate route). Matches the Articles / Journals / Live Sessions list pattern exactly.

Refs: .cms-table · .content-block · row-clickable hover · .pill · .val-pill

V2 — Stacked content-blocks
Each series is its own content-block with parts inline

Series stack vertically as .content-blocks. Block header carries name + parts count + Live button. Body shows description, then a rowform list of parts (same chrome as Navigation). Add-part row at the bottom. Everything visible at a glance, no extra navigation.

Refs: .content-block · .rowform-row · .pill · .btn-sec · .btn-icon-danger

V3 — Master-detail (two-pane)
Sidebar list + detail panel

Left rail shows all series as a compact list. Click one to open it in the right panel — same affordance as Post Templates view (the only existing two-pane in the CMS). Scales well when there are 10+ series.

Refs: Post Templates two-pane · .content-block · .rowform-row

V4 — Restructured card grid
Card grid, but each card uses the canonical block chrome

Keeps the at-a-glance grid format but rebuilds each card to look like a mini content-block: bordered header (label + sublabel), body with description + parts preview, footer with actions. Fixes the "feels different" complaint while preserving the dashboard energy.

Refs: .content-block header/body/foot · mini-rowform · .pill

V5 — Rowform with inline expand
Each series is a single rowform-row; chevron expands to show parts

Most compact option. Each series is one row of a rowform list (same chrome as Navigation) showing name, slug, parts count, status. Click the chevron to expand — parts list and add-form drop down inline. Multiple can be open at once or just one (your choice).

Refs: .rowform-row (with inline expand) · .pill · matches Navigation visually

V6 — Hybrid: list + sticky detail strip
Compact rowform list, with one expanded "active" series under a sticky strip

A compromise of V3 and V5: top half is a compact rowform list (like Navigation). Below it, the currently-selected series gets its own full .content-block with parts inline. Click another row to swap. Combines scan-ability with full editing in one screen.

Refs: .rowform-row · .content-block · .pill
