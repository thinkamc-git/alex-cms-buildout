There's a tension every designer feels right now. The tools are getting faster. The work is getting harder.

For a decade, the discipline trained itself to value execution — pixel-perfect mocks, clean component libraries, design tokens that compile into code. AI is collapsing that surface area in months. What it can't do, at least not yet, is the part of design we've under-invested in: judgment.

Content Template
Each content type uses a PHP layout file that controls how its fields render on the live site. The Master template lists every available field and its PHP variable — it doesn't turn anything on or off. Each sub-template inherits everything and can suppress specific fields. The Master layout file is the most complete version; sub-templates only remove things from it.
Reference
Master Template
Defines every block, field, and the author profile. Each sub-template inherits all of this — sub-templates only suppress optional blocks.
article: standard system
Default long-form article. Body is a Tiptap rich text field.
article: series system
Articles that belong to a series. Series field required; prev/next nav auto-renders.
journal entry system
Short reflective entries. Key Statement replaces Title; Entry Number auto-assigned.
live session system
Talks, workshops, masterclasses. Includes Event Details and Format Tags.
experiment system
Article-format experiment. Uses the rich text Body — no HTML import.
experiment: html system
Raw HTML imported from a folder. For prototypes and hand-built pages.

PHP Layout File
The Master Template defines every block available across content types. Each block has a stable slug used in code (data-block) and a visibility mode. Always blocks render whenever applicable. Optional blocks are toggled on or off per content type from each sub-template. Auto blocks render based on the data (e.g. Tags renders only when tags exist). To change what appears on a sub-template, select it from the list on the left.
